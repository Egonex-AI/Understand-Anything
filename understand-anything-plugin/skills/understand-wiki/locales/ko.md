# 한국어 Wiki 출력 가이드 (Korean)

이 파일은 한국어로 Wiki 문서를 생성하기 위한 언어 지침을 제공합니다.

## 이름 번역 규칙

모든 `name` 필드는 한국어로 작성해야 합니다 (`id` 필드는 영어 유지):

| 필드 | 영어 (DG 출처) | 한국어 (출력) |
|---|---|---|
| Domain `name` | `"Order Management"` | `"주문 관리"` |
| Flow `name` | `"Create Order"` | `"주문 생성"` |
| Step `name` | `"Validate Input"` | `"입력 검증"` |
| Entity `name` | `"Order"` | `"Order"` (영어 클래스명 유지) |

- Domain name: 비즈니스 도메인 개념 번역
- Flow name: 동사구 (예: "주문 생성", "결제 처리", "재고 동기화")
- Step name: 간결한 동작 설명 (예: "입력 검증", "가격 계산", "데이터 영속화")
- Entity name: 코드의 클래스명(영어) 유지, 번역하지 않음

## 요약 스타일

- 도메인 요약: 3-5문장의 한국어로 비즈니스 능력, 핵심 엔티티, 주요 규칙 및 외부 종속성 설명
- 플로우 요약: 2-3문장의 한국어로 비즈니스 목표, 기술 메커니즘 및 크로스 서비스 상호 작용 설명
- 단계 설명: 코드가 **무엇을 하는지** 설명 (**무엇인지**가 아니라)

## 유비쿼터스 언어

- 코드의 클래스명/열거형명을 용어(`term`)로 사용하고, 정의는 한국어로 작성
- 각 도메인당 5-15개 용어를 목표

## 비즈니스 규칙

- 규칙 설명은 한국어로 작성
- `enforcement` 필드는 코드의 클래스명/메소드명을 그대로 사용

## 기술 용어

다음 용어는 영어를 유지:
- `DTO`, `VO`, `Entity`, `Repository`, `Service`
- `Kafka`, `Redis`, `MySQL`, `Elasticsearch`
- `RPC`, `Dubbo`, `gRPC`, `REST API`
